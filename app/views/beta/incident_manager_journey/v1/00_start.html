Overview of incident manager review (linear journey)

Notes:

  • This email is triggered by the confirmation page of the incident report journey
  • This prototype links straight to the first question of the journey: in reality, the journey will start with Searchlight sign in and a dashboard screen (TBC)
  • There are three versions of content for this email. Which one the incident manager sees will depend on what the incident reporter selected on the 'Identify appropriate manager' screen:
  1. If the incident reporter selected 'My line manager' on that screen to indicate that the line manager and the incident manager are the same person, email 1 shows the content for the email that should go to the line / incident manager.

  2. If the incident reporter selected 'Another manager' on that screen to indicate that the line manager and the incident manager are different people, email 2 shows the content for the email that should go to the line manager.

  3. If the incident reporter selected 'Another manager' on that screen to indicate that the line manager and the incident manager are different people, email 3 shows the content for the email that should go to the incident manager.
  • Email 1, 2 and 3 are designed here but not finalised
  • Email 4, 5 and 6 will have similar content to 1, 2 and 3, to re-iterate advice. We've provided a low fidelity prototype for context.
  • This prototype has a "Witness details not recognised" page to be used when the witness is not a DWP employee. However, the first cut of this journey does not need this page because we are assuming that we will use a manual process. This may change in the future. The page can be accessed in the journey flow by putting "Mike Black" in the "Full name" field on the "Enter witness details" page.
